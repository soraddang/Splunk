Switch 특징 
- MAC address table
- forwarding/ flooding 데이타 전송 
( 수신지 mac )이  ( mac address table  )에 있으면 포워딩
( 수신지 mac )이  ( mac address table   )에 없으면 플러딩 

Router 특징 
- Routing table
- forwarding 방식으로만 데이타 전송 
( 수신지 IP 또는 수신 네트워크 ID )가  ( 라우팅테이블  )에 있으면 포워딩
( 수신지 IP 또는 수신 네트워크 ID )이  ( 라우팅테이블  )에 없으면 폐기(drop) 

switch/router 공통점
- 경로데이타베이스에 멀티캐스트/브로드캐스트 주소를 등록하지 않음

멀티캐스트/브로드캐스트 처리 방식 
 - switch : 플러딩 
 - router : 폐기


허브 
 - 1계층
 - 경로데이타베이스 없음
 - 전송방법 : 플러딩 

스위치
 - 2계층
 - 경로데이타베이스 : MAC address table
 - 전송방법 : 플러딩, 포워딩
   ( Multicast/Brodcast 패킷 처리 방식 : 플러딩)

라우터 
 - 3계층
 - 경로데이타베이스 : Routing table
 - 전송방법 : 포워딩
   ( Multicast/Brodcast 패킷 처리 방식 : drop)

Switch 종류
   L2 SW = L1 + 전달기능 
   L3SW = L2 SW + 라우팅 기능 
   L4SW = L3 SW + @
   L7SW = L4 SW   +@ 


Router  : 장비 
Routing : 최적 경로를 선정하는 작업 
Route : 최적 경로


L2SW 
 - 수신지 MAC address
 - Fowarding(전달)==>Switching  

L3SW 
 - 수신지 IP address
 - Routing 
 
L4SW 
 - 수신지 IP address + 수신지 Port번호
 - Load balancing (트래픽 균일 부하) ==> 로드 밸런서

L7SW 
 - 수신지 IP address + 수신지 Port 번호 + Data
 - Load balancing (트래픽 균일 부하) ==> 로드 밸런서
 - Load sharing (트래픽 비균일부하)

IP address = NetworkID+HostID
